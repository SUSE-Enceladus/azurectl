Information about the commands and its usage from the Microsoft
azure nodejs based commandline utility. We keep this data as
implementation reference and for backward compatibilty with the
azurectl python application

---------------------------------------------------
Launch a browser to download your publishsettings file
Usage: account download [options]
    
    Options:
      -h, --help                       output usage information
      -v, --verbose                    use verbose output
      --json                           use json output
      -e, --environment <environment>  the publish settings download environment
      -r, --realm <realm>              the organization's realm

---------------------------------------------------
Remove a subscription or environment, or clear all of the stored account and environment info
Usage: account clear [options]
    
    Options:
      -h, --help                                output usage information
      -v, --verbose                             use verbose output
      --json                                    use json output
      -s --subscription <subscriptionNameOrId>  Subscription name or id to remove
      -e --environment <environmentName>        Environment name to remove
      -q --quiet                                quiet mode, do not ask for delete confirmation

---------------------------------------------------
Exports the publish settings file as a PEM file
Usage: account cert export [options]
    
    Options:
      -h, --help                               output usage information
      -v, --verbose                            use verbose output
      --json                                   use json output
      -f, --file <file>                        the name of the cert file. If not specified, generate a file in pwd using the subscription ID as the file name
      -p, --publishsettings <publishsettings>  the publish settings file
      --subscription <subscription>            the Name or ID for the subscription whose cert you want to export. If not specified, use the current subscription

Create a cluster
Usage: hdinsight cluster create [options] [config]
    
    Options:
      -h, --help                                 output usage information
      -v, --verbose                              use verbose output
      --json                                     use json output
      --config <config>                          the config file for cluster creation
      --clusterName <clusterName>                the HdInsight cluster name
      --storageAccountName <storageAccountName>  the storage account to use for HDInsight storage
      --storageAccountKey <storageAccountKey>    the key to the storage account to use for HDInsight storage
      --storageContainer <storageContainer>      the container in the storage account to use for HDInsight default storage
      --nodes <nodes>                            the number of data nodes to use for the cluster
      --location <location>                      the data center location for the cluster
      --username <username>                      the user name to use for the cluster
      --clusterPassword <clusterPassword>        the password to use for the cluster
      -s, --subscription <id>                    the subscription id

    Options:
      -h, --help                               output usage information

      --type <metastoreType>                   the type of metastore to set (example: hive, oozie)

      --database <database>                    the name of the database on the sql server
      --user <userName>                        the user name to use when connecting to the sql server
      --metastorePassword <metastorePassword>  the password to use when connecting to the sql server
